The Gap

AI agents are running in production with shared API keys, static credentials, and no per-action enforcement. When something goes wrong, three questions go unanswered:

Identity
Which agent acted — on whose behalf, under what authorization?
Authorization
Was this specific action in policy at the moment it ran?
Proof
Can you prove it without trusting the party under investigation?

Existing NHI platforms scope credentials at provisioning time. Nobody enforces policy at execution time, per action. When an incident occurs, the forensic trail is unsigned app logs — controlled by the party under investigation.

Identity and access are the first problem. Evidence is the exhaust. AgentMint is the gateway where agents get identities, policies are enforced per action, and cryptographic receipts are generated as a byproduct of that decision — not a separate logging step.

What AgentMint Does

01
Issue Agent Identities
Each agent gets a cryptographic identity scoped to the task — tied to owner, purpose, and risk tier. Not a shared API key. Not a static credential.
02
Enforce Least-Privilege
Every action evaluated against policy at execution time — not provisioning time. Blocked actions never reach the tool. Supports checkpoints for sensitive operations.
03
Emit Cryptographic Receipts
Every allowed or denied action generates a tamper-evident receipt:
  • Agent identity + policy version
  • Decision (allow / deny) + content hashes
  • Ed25519 signature + RFC 3161 timestamp
  • AIUC-1 / ISO 42001 control mapping
04
Package Evidence
Receipts accumulate in an append-only store, exportable as evidence packs for AIUC-1 recertification, ISO 42001, EU AI Act, and insurance underwriting.

Architecture — Three Independent Anchors

No single party — including AgentMint — can tamper with the evidence chain. Private key never leaves the customer environment. If AgentMint disappears, every receipt remains verifiable with OpenSSL alone.

Anchor 01
Ed25519 Signature
Private key generated on customer machine. Never transmitted. Any field change breaks the signature. Verifiable with OpenSSL alone — no AgentMint dependency.
Anchor 02
RFC 3161 Timestamp
SHA-512 hash sent to FreeTSA — independent of both AgentMint and the customer. Backdating is mathematically impossible. Only a hash leaves the environment.
Anchor 03
Commitment Scheme
Receipts contain hashes of evidence, not evidence itself. No sensitive content leaves the customer environment. CA certs and public key included in the zip.

How to Engage

The prototype is open-source, MIT-licensed, and generating real receipts against production APIs. Looking for a small set of design partners:

  • AI-native SaaS companies shipping agents into enterprise production.
  • Enterprises pursuing AIUC-1 / ISO 42001 / EU AI Act readiness.
  • Insurers and auditors experimenting with cryptographic evidence formats for AI risk.
Proposed

Run AgentMint alongside one real agent in a non-critical workflow. Compare the identity and evidence story against whatever you have today when an incident or audit comes around. Zero availability risk — never touches existing integrations. No data leaves except a SHA-512 hash to FreeTSA.
