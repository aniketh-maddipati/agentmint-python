AgentMint

The problem

AIUC-1 certification today is a point-in-time attestation. Between quarterly tests, enterprises have no independent proof their AI agent calls were within policy. Evidence assembly takes 3–5 weeks and repeats every quarter. When an incident occurs at month two, the forensic trail is unsigned app logs — controlled by the party under investigation.

AgentMint makes certification continuously provable. Every agent action generates a tamper-evident, cryptographically signed receipt — accumulating passively in the background until the auditor runs VERIFY.sh.

Architecture — three independent anchors

AgentMint never wraps or proxies API calls. It observes responses after they return. No single party — including AgentMint — can tamper with the evidence chain.

Anchor 01
Ed25519 Signature
Private key generated on customer machine. Never transmitted. Any field change breaks the signature. Verifiable with OpenSSL alone — no AgentMint dependency.
Anchor 02
RFC 3161 Timestamp
SHA-512 hash sent to FreeTSA — independent of both AgentMint and the customer. Backdating is mathematically impossible. Only a hash leaves the environment.
Anchor 03
Commitment Scheme
Receipts contain hashes of evidence, not evidence itself. No sensitive content leaves the customer environment. If AgentMint disappears, every receipt remains verifiable. The CA certs and public key are in the zip.

The ask

The technology works. The question is whether evidence format standardization is a real operational gap — or whether enterprises are already solving this in ways that aren't visible from outside.

Proposed

Run AgentMint as a passive sidecar during one real quarterly review cycle. At quarter-end, compare this evidence package against whatever your team currently produces. Zero availability risk — never touches existing API integrations. No data leaves except a SHA-512 hash to FreeTSA.
